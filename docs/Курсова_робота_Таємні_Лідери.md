## Розділ 1. Теоретичні засади веб-додатків та технологій проєкту

### 1.1. Призначення та короткий опис настільної гри «Таємні Лідери»

**«Таємні Лідери»** — стратегічна гра з прихованими фракціями, лідерами та картковими механіками. У цифровій реалізації користувач може **створити кімнату** (обравши кількість гравців 2–6) або **приєднатися за кодом кімнати** (чотиризначний ідентифікатор). Після збору гравців у лобі можливий **старт партії**; під час гри реалізовані фази ходу (наприклад, гра картою, добір, скидання, поповнення «таверни» тощо згідно з правилами, закладеними в доменній логіці).

### 1.2. Архітектура «клієнт — сервер» та REST API

У проєкті застосовано класичну **клієнт-серверну** модель:

- **Сервер** (Python, **FastAPI**) зберігає події гри, керує кімнатами, перевіряє команди гравців і видає стан з урахуванням «видимості» для кожного гравця.
- **Клієнт** (браузер, **Next.js** + **React**) відображає інтерфейс і викликає HTTP API з префіксом `/v1`, передаючи при потребі заголовок **`X-Player-Token`** для ідентифікації гравця в кімнаті.

Такий підхід дозволяє масштабувати сервер окремо від інтерфейсу та підключати інших клієнтів за тим самим API.

### 1.3. Обраний стек технологій

| Компонент | Технології |
|-----------|------------|
| Бекенд | Python 3, **FastAPI**, Pydantic, подієве сховище (event store), SQLite (ініціалізація через `init_db`) |
| Доменна логіка | модулі `domain/` (стан, редʼюсер подій, проєкція, обробники команд) |
| Фронтенд | **Next.js 14** (App Router), **React 18**, **TypeScript**, **Tailwind CSS** |
| Дані карт | `data/cards.json` (каталог завантажується при старті сервера) |

Фронтенд звертається до базової адреси API змінної середовища **`NEXT_PUBLIC_API_URL`** (за замовчуванням `http://localhost:8000`).

## Розділ 2. Структура програмного забезпечення

### 2.1. Репозиторій та модулі бекенду

Корінь репозиторію містить довідкові або legacy-скрипти (`main.py`, `game.py` у корені — для консольних прототипів). Основний **серверний застосунок** розташований у каталозі **`src/back/`**:

- **`app.py`** — додаток FastAPI, маршрути `/v1/...`, CORS, моделі запитів;
- **`session.py`**, **`event_store.py`** — сесії кімнат і збереження подій;
- **`command_handlers.py`** — обробка ігрових команд;
- **`domain/`** — чиста ігрова логіка: команди, події, стан, умови, завершення гри;
- **`run.py`** — точка входу для запуску сервера;
- **`tests/`** — unit- та інтеграційні тести.

### 2.2. Фронтенд: маршрути та основні екрани

Каталог **`src/front/`** — проєкт Next.js:

- **`app/page.tsx`** — головний екран: введення імені, створення кімнати або вхід за кодом, посилання на правила (модальне вікно);
- **`app/room/[roomId]/page.tsx`** — кімната / лобі / ігровий екран для обраної кімнати;
- **`components/`** — перевикористовувані компоненти UI;
- **`lib/api.ts`** — обгортки над `fetch` до REST API (`createRoom`, `joinRoom`, `getState`, команди тощо).

### 2.3. Авторизація гравця та зберігання стану в браузері

Після успішного **`join`** сервер повертає ідентифікатори гравця та **токен**. Фронтенд зберігає їх у **`localStorage`** (зокрема ключі виду `hl_token_<room_id>`, `hl_player_<room_id>`, `hl_name_<room_id>`), щоб після оновлення сторінки гравець залишався в сесії. Запити до захищених ендпоінтів супроводжуються заголовком **`X-Player-Token`**.

## Розділ 3. Реалізація та експлуатація

### 3.1. Запуск бекенду та фронтенду

**Бекенд** (з каталогу `src/back`):

```bash
python run.py
```

Сервер очікує з’єднання (типово порт **8000**; уточніть у `run.py` / конфігурації).

**Фронтенд** (з каталогу `src/front`):

```bash
cp .env.local.example .env.local
# У .env.local задати NEXT_PUBLIC_API_URL=http://localhost:8000
npm install
npm run dev
```

Відкрити в браузері **http://localhost:3000**. Спочатку має бути запущений бекенд.

У Windows у корені репозиторію можуть бути скрипти **`start-backend.bat`**, **`start-frontend.bat`** — для швидкого запуску без ручного набору команд.

### 3.2. Ігровий цикл і команди (API)

Клієнт послідовно:

1. **`POST /v1/rooms`** — створення кімнати (`num_players`, `game_mode`);
2. **`POST /v1/rooms/{room_id}/join`** — приєднання з ім’ям;
3. **`POST /v1/rooms/{room_id}/start`** — старт гри (з токеном);
4. **`GET /v1/rooms/{room_id}/state`** — поточний стан для гравця;
5. команди ходу (згідно з `command_handlers.py` та доменними типами): гра картою, пас, скидання карт, добір з таверни/гавані, поповнення таверни тощо.

Деталі відповідей узгоджені з моделями у `app.py` та типами в `domain/commands.py`.

### 3.3. Тестування

У **`src/back/tests/`** наявні тести редʼюсера, проєкції, обробників команд, умов завершення гри та інтеграційний сценарій API. Запуск (залежно від середовища):

```bash
cd src/back
pytest
```

Це підвищує надійність змін у доменній логіці.

## Розділ 4. Рекомендації з розгортання та подальшого розвитку

1. **Production**: винести API за HTTPS, обмежити CORS у `config.py` конкретними доменами, використовувати змінні середовища для секретів і URL.
2. **Фронтенд**: `npm run build` та `npm run start` для продакшен-збірки Next.js.
3. **Моніторинг**: логування вже налаштоване через `logging`; варто додати ротацію логів на сервері.
4. **Розвиток**: розширення каталогу карт, покращення UX кімнати та анімацій, локалізація інтерфейсу.

## Висновки

У ході підготовки документації розглянуто веб-проєкт **«Таємні Лідери»** як **клієнт-серверну систему** з бекендом на **FastAPI** та фронтендом на **Next.js**. Описано подієву модель збереження стану, основні каталоги коду, сценарії створення кімнати та приєднання гравця, а також порядок локального запуску. Проєкт демонструє застосування сучасних підходів (REST API, TypeScript, компонентний UI, тести домену) до реалізації настільної гри в браузері.
